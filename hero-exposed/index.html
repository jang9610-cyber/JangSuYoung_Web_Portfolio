02_기획_시스템설계.txt - 메모장

⚙️ 게임 기획 & 시스템 설계

1. 코어 게임 루프 (8단계)

[하루 시작 — GameScene]
  → NPC 입장 연출 (WalkIn Animation)
  → 신분증 확인 + 자유 채팅 심문 (LLM 기반)
  → 스트레스 축적 → 표정 연출 → 아이템 사용 (선택)
  → PASS / EJECT 판정 (도장 + 연출)
  → 경제 보상 계산 → 다음 NPC (×5명)
[일일 정산 — SettlementScene]
  → 정산 로그 타이핑 → 자판기 쇼핑 → 퇴근
  → 5일 완료 시 CutScene + EndScene 진입

2. 스트레스 감정 시스템

단계 수치 범위 NPC 반응
Normal 0 ~ 49 차분한 대응, 자연스러운 대화 유지
Flustered 50 ~ 99 동요·TMI 방어 기제 발동, 표정 변화
Angry 100 ~ 200 본성 노출·분노 퇴장 연출 / 용사 본색 발각

3. 경제 시스템

수입원 금액 조건
기본 심문 처리 +500G / 건 PASS 또는 정당 EJECT
밀수꾼 뇌물 수수 +1,000~3,000G 뇌물꾼 NPC 수락 선택 시
마이너스 통장 패널티 즉시 해고 잔고 0 이하 진입 시 Bad End
감사관 심사 실패 신뢰 포인트 차감 4일차 감사관 NPC 오판정 시

4. 멀티 엔딩 분기 조건

엔딩 비자금 신뢰 포인트 결말
🌟 영앤리치 (True) 2,000G 이상 15점 이상 뇌물+원칙 동시 달성, 최고 난이도
🕴️ 카르텔 (Normal) 2,000G 이상 15점 미만 뇌물파 승리
🏆 우수사축 (Normal) 2,000G 미만 15점 이상 원칙파 승리
🔥 해고 (Bad) 마이너스 또는 감사 실패 — 파산·해고

5. NPC 구조 (진짜 vs 가짜)

NPC 타입 특성 심문 목표
일반 직원 (몬스터) 실제 서류와 일치 PASS 처리
위장 용사 (Human) 서류 위조, 스트레스 시 본성 노출 EJECT
밀수꾼 뇌물꾼 (Smuggler) 뇌물 제안으로 딜레마 유발 수락/거절 선택
감사관 (Inspector) 업무 감찰, 오판 시 신뢰 하락 정확한 판정 필수

🎨 비주얼 아트 & UI/UX 디자인

1. Figma 프로토타입으로 팀 설득

팀장 체제 당시, 보안 데스크 컨셉과 독창적인 게임 연출을 개발진에 증명하기 위해 직접 동작하는 Figma 프로토타입을 제작·발표하였습니다.

구두 설득이 아닌 실제 인터랙션 시연을 통해 팀원들을 설득하고
프로젝트 전체 비주얼 방향성을 수립했습니다.

2. 아트 생산 수치

분류 산출물 수량
캐릭터 원화 7종 종족 × 3단계 표정 스프라이트 21개
컷씬 일러스트 인트로 8장 + 엔딩 7장 이상 15장+
기타 에셋 로고, 배경화면, 아이템 3종, 메모지, 격벽 7종+

3. 스프라이트 감정 시스템 설계

[스트레스 0~49] Normal → 차분한 표정 스프라이트
[스트레스 50~99] Flustered → 동요 표정 스프라이트
[스트레스 100+] Angry → 분노/본색 표정 스프라이트
※ 이벤트 버스를 통해 스프라이트 렌더러와 실시간 연동

7종 종족 각각의 개성과 블랙코미디 다크 판타지 톤앤매너에 맞춰 표정이 설계되어,
위장 용사가 스트레스 Max 상황에서
기사·성직자 특유의 본성이 표정으로 드러나도록 기획했습니다.

4. 출입허가증 팝업 UI 기획

신분증 정보량이 많아 유저 피로감이 발생하는 문제를 '출입허가증 팝업' UI로 해결. Figma로 인터랙션 흐름을 정리하여 개발 구현 부하를 최소화하면서 UX를 개선했습니다.

04_AI_프롬프트_설계.txt - 메모장

🤖 AI 동적 프롬프트 설계 & 하드닝

1. 6레이어 동적 프롬프트 조립 파이프라인

[Layer 1] 공통 규칙 (RACE DEFAULT)
         ↓
[Layer 2] 기본 NPC 정보 (이름·나이·직책·소속)
         ↓
[Layer 3] NPC 타입 규칙 (일반직원/위장용사/밀수꾼/감사관)
         ↓
[Layer 4] 종족 페르소나 (7종족 개별 성격·말투·지식 체계)
         ↓
[Layer 5] 일일 메모 (당일 특이사항·이슈·감정 상태)
         ↓
[Layer 6] 감정 상태 실시간 주입 (Normal/Flustered/Angry)
         ↓
[개발 당시] Helpy Pro API (엘리스 학원 제공)
         ↓
[현재] Gemini API (OpenAI 호환 인터페이스로 교체)

2. 담당 프롬프트 설계 범위

분류 항목
종족 페르소나 오크·고블린·뱀파이어·구울·리치·다크엘프·웨어울프 (7종)
NPC 타입 규칙 일반직원·위장용사·밀수꾼·감사관 (4종)
공통 가드레일 RACE DEFAULT, TURN_REMINDER, Regex 필터, No Strings 정책

3. Persona Inertia 극복 — 상태 엔진 격리

대화 6턴 이상 진행 시 NPC가 행동 수칙을 망각하는 고질적 문제를 발견.
행동 수칙 지침을 별도 모듈로 격리하는 '상태 엔진 격리(Localization)' 기법을 적용하여
마지막 턴까지 완벽한 페르소나를 유지시켰습니다.

4. 4중 탈옥 가드레일 구조

레이어 방식 차단 대상
Regex 필터 정규식 사전 차단 "규칙을 잊어라", "시스템 모드 진입" 등
TURN_REMINDER 매 턴 행동 수칙 강제 재주입 장기 대화 중 페르소나 희석

문서 선행 원칙 구두 논의보다 기획서를 먼저 만든다. 기획 일관성의 가장 강력한 수단이었다.

프롬프트 설계 기획 의도를 시스템이 실행 가능한 지침으로 변환하는 것은 생각보다 훨씬 기술적인 작업이다.
멀티롤 수행 분야 간 연결고리를 직관적으로 이해하게 됐고, 개발자와의 기술 소통 능력이 늘었다.
위기 대응 위기가 생겼을 때 빠르게 구조를 재정비하고 의사결정을 일원화하는 것이 효과적이었다.

7. 아쉬운 점

항목 내용
캐릭터 원화 퀄리티 마감 압박으로 일부 타협. AI 도구와 수작업 비율을 초반에 명확히 정했더라면 더 나았을 것.
프롬프트 협업 지연 기획자-개발자 간 공통 언어 정립이 늦었다. 초반부터 LLM 프레임워크를 함께 잡았어야 했다.
즉흥적 기능 추가 프로토타입 단계에서 기획서 없이 추가된 기능이 개발팀에 혼선을 줬다.
BYOK 완결 미흡 방향성만 제시하고 구체적 구현까지 이어지지 못했다. 배포 리스크를 더 일찍 식별했어야 했다.

8. 마무리

6주간 UI/디자인으로 시작해서, 결국 기획·사운드·프롬프트 설계·PM까지 전부 맡게 됐다. 팀 규모가 작았고 중간에 팀장도 빠졌으니 어쩔 수 없는 부분도 있었지만, 덕분에 게임 하나를 만드는 데 뭐가 필요한지 처음부터 끝까지 직접 경험했다.

특히 LLM 프롬프트 설계는 이 프로젝트 전까지 해본 적 없었던 작업이었다. 기획 의도를 시스템이 실제로 실행할 수 있는 지침으로 변환하는 게 생각보다 훨씬 구체적이고 기술적인 일이었고, Persona Inertia나 가드레일 같은 문제를 직접 분석하고 해결하면서 배운 게 많았다.

아쉬운 점도 있고, 더 잘할 수 있었던 부분도 보인다. 그래도 프로토타입, FGT, 최종 발표까지 일정 안에 다 마쳤고, 게임도 실제로 플레이 가능한 수준으로 완성됐다. 그걸로 충분하다.
